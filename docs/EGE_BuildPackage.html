Final API Map

RESTful /api/v1/ · Bearer JWT · Company isolation at middleware · SSE for AI Chat · Cursor pagination

/api/v1/auth — Authentication
POST/auth/registerRegister new user + create company → JWT pair
POST/auth/loginEmail/password → access token + refresh token
POST/auth/refreshRotate refresh token → new access token
POST/auth/inviteSend email invitation to join company
POST/auth/accept-invite/:tokenAccept invitation + join company with specified role
GET/auth/meCurrent user profile + active company context + role
/api/v1/diagnostics — Diagnostic System (M2)
POST/diagnostics/cyclesStart new diagnostic cycle (only one active allowed)
GET/diagnostics/cyclesList all company cycles with status + completion date
GET/diagnostics/cycles/:idGet cycle with all responses + progress
POST/diagnostics/cycles/:id/responsesSubmit responses for one or more dimensions (idempotent)
POST/diagnostics/cycles/:id/completeComplete cycle → trigger scoring job → trigger AI analysis job
GET/diagnostics/scores/:cycleIdGet dimension scores + benchmark deltas + maturity level
GET/diagnostics/snapshotsGet full maturity history (all completed cycles)
/api/v1/plans — Strategic Planning (M3)
POST/plans/cyclesCreate quarterly strategic cycle
GET/plans/cycles/:idGet cycle with all objectives + KRs + initiatives
POST/plans/objectivesCreate objective linked to cycle + dimension
POST/plans/key-resultsCreate key result with metric + target
PATCH/plans/key-results/:idUpdate KR current value → auto-recalculate objective health
POST/plans/generateAI-generate full OKR plan from latest diagnostic → async job
GET/plans/generate/:jobIdPoll AI plan generation job status + result
/api/v1/execution — Execution (M4)
GET/execution/tasksList tasks — filter: assignee, status, initiative, due date
POST/execution/tasksCreate task linked to initiative
PATCH/execution/tasks/:idUpdate task status, assignee, due date
POST/execution/kpisCreate KPI config (name, target, unit, frequency, owner)
POST/execution/kpis/:id/valuesSubmit KPI value for period (append-only)
GET/execution/kpis/:id/historyGet KPI time series with trend + target comparison
POST/execution/check-insSubmit weekly or monthly check-in
/api/v1/ai — AI Intelligence (M5)
POST/ai/chatSend message to AI Advisor → SSE stream (text/event-stream) with context-injected response
GET/ai/recommendationsList active recommendations (filter by priority, status, type)
PATCH/ai/recommendations/:id/statusAcknowledge / dismiss / complete recommendation
POST/ai/recommendations/:id/feedbackSubmit feedback → rating + action_taken → C8 learning signal
GET/ai/alertsGet pending alert queue with priority + channel status
GET/ai/contextGet current AI context snapshot (diagnostic, KPIs, OKRs, benchmarks)
API Design Rules
RuleImplementation
AuthenticationBearer JWT; company_id extracted from JWT claims — never from request body
Rate Limiting100 req/min per user (Redis-backed). 10 req/min for /ai/* endpoints.
VersioningURL prefix /api/v1/ — breaking changes require /v2/ — no header versioning
PaginationCursor-based: ?cursor=&limit= on all list endpoints — no offset pagination
Async JobsPOST returns { jobId }. GET /jobs/:id polls { status, progress, result }.
ErrorsRFC 7807: { error: { code, message, details } } — consistent across all endpoints
Internal CommsService-role key in Authorization header for M9 + M5 workers — never client-accessible

Recommendation Scoring Formula
score = (impact × 0.35) + (speed × 0.25) + (evidence × 0.25) + (efficiency × 0.15)
impact — business gap × benchmark delta
speed — time-to-result (inverted)
evidence — data confidence 0–1
efficiency — resource cost (inverted)
Threshold: score < 0.60 → suppressed. Never shown to user.
