Comando 02
Banco de Dados PostgreSQL
Schema multi-tenant com RLS, índices estratégicos e isolamento por empresa
Princípios do Schema
🔑 UUID como PK
Evita enumeração de registros e facilita sharding horizontal futuro.
🏢 company_id universal
Presente em todas as tabelas de negócio — chave principal de isolamento multi-tenant.
🔒 RLS nativo
Row Level Security no PostgreSQL garante que cada empresa só acessa seus dados, mesmo sem validação no código.
🗑 Soft Delete
Campo deleted_at em todas tabelas — preserva histórico e permite recuperação de dados.
Tabelas Principais

Auth Middleware + RLS Injection
export const authMiddleware = async (req, res, next) => { const token = req.headers.authorization?.split(' ')[1]; if (!token) return res.status(401).json({ error: 'Unauthorized' }); const payload = verifyJWT(token); req.userId = payload.userId; req.companyId = payload.companyId; // Injeta company_id no PostgreSQL para ativar RLS await db.query(`SET app.company_id = '${payload.companyId}'`); next(); };

Comando 07
AI Business Advisor
Como a IA analisa dados reais e gera planos de crescimento acionáveis para cada empresa
Fluxo de Análise (7 Etapas)
01
Evento Disparador
Diagnóstico concluído / nova métrica / lead parado 7d+ / solicitação manual
02
Coleta de Contexto
Busca no PostgreSQL: empresa, diagnóstico, pipeline, métricas 6 meses, tarefas, recomendações anteriores
03
Montagem do Prompt
JSON estruturado com todo contexto + system prompt de consultor sênior especializado no setor
04
Chamada Claude API
claude-sonnet-4-20250514 · max_tokens: 2048 · temperature: 0.3 · resposta em JSON estruturado
05
Parse e Classificação
Extrai type, title, content, priority, action_items[] da resposta JSON da IA
06
Persistência
Salva em ai_recommendations com status "active", vinculada à empresa e ao evento de origem
07
Notificação
n8n dispara alerta no dashboard (realtime) + email com resumo das recomendações geradas
