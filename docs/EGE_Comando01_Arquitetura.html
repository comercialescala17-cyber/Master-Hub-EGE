01Stack por Camadas — Da IA ao Usuário
Camada 1 — IA & Automação
O motor inteligente da plataforma. Claude API processa diagnósticos e gera recomendações estratégicas. n8n orquestra todos os fluxos assíncronos sem código. OpenAI fornece embeddings para busca semântica.
Claude APIn8n WorkflowsOpenAI EmbeddingsWebhooksEmail Service (Resend)
Camada 2 — Dados & Persistência
PostgreSQL com Row Level Security garante isolamento total entre empresas. Redis para cache de sessões e respostas frequentes da IA. Supabase Storage para arquivos e relatórios. Backups automáticos diários.
PostgreSQL 15Supabase (BaaS)Redis CacheRow Level SecurityS3-compatible Storage
Camada 3 — Serviços de Negócio
Microserviços independentes por domínio. Cada serviço possui sua própria lógica, testes e pode escalar separadamente. Comunicação via eventos internos para operações assíncronas.
Auth ServiceCompany ServiceDiagnostic ServiceAI ServiceCRM ServiceAnalytics ServiceTask Service
Camada 4 — API Gateway
Porta de entrada única para o frontend. Valida JWT, injeta company_id no contexto, aplica rate limiting por tier de plano, e roteia para o serviço correto. Logs centralizados de todas as requisições.
Node.js + ExpressJWT MiddlewareRate LimitingRequest LoggerCORS + Helmet
Camada 5 — Frontend
SPA React com roteamento client-side. React Query gerencia cache de dados e sincronização automática. Zustand para estado global leve. Recharts para visualizações. Design system próprio com Tailwind.
React 18Tailwind CSSReact QueryZustandRecharts (Radar + Line)React DnD (Kanban)
Camada 6 — Usuário Final
Interface acessada via browser ou PWA mobile. Experiência fluida independente do dispositivo. Atualizações em tempo real via Supabase Realtime para notificações e mudanças no pipeline.
Web BrowserPWA MobileSupabase RealtimePush Notifications

04Como Escalar para Milhares de Empresas
TIER 1
0→1K
empresas ativas
Infraestrutura Inicial
Supabase Free/Pro — DB + Auth + Storage centralizados
Railway ou Render — 1 instância Node.js com auto-restart
n8n Cloud — workflows sem gerenciar servidor
Redis compartilhado para cache de sessões
Custo estimado: R$ 300–600/mês
TIER 2
1K→10K
empresas ativas
Escala Horizontal
PostgreSQL dedicado com read replicas para analytics
2–4 instâncias Node.js atrás de load balancer
Redis Cluster para cache distribuído
n8n self-hosted com worker nodes separados
CDN para assets estáticos (CloudFlare)
Custo estimado: R$ 2K–5K/mês
TIER 3
10K+
empresas ativas
Arquitetura Enterprise
Sharding por região (Sul, Sudeste, Nordeste)
Microsserviços em containers (Kubernetes / ECS)
Message queue (SQS ou RabbitMQ) para picos de IA
Metabase dedicado por cluster de clientes enterprise
SLA 99.9% com failover automático
Custo estimado: R$ 15K–40K/mês
