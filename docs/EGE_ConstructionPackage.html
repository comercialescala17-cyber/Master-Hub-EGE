Key Design Decisions

DecisionImplementation
JWT StrategyShort-lived access token (15min) + long-lived refresh token (7d). Rotation on refresh.
Multi-companyOne user can belong to multiple companies. companyId in JWT claims — switched on login.
RLS Enforcementcompany_id injected into req.context by middleware before any resolver. Prisma RLS via Supabase.
Audit LogEvery write action writes to audit_logs. Append-only. No UPDATE or DELETE ever granted.

Scoring Algorithm

StepLogic
1. Normalizescore 1–5 → 0–100: ((score - 1) / 4) × 100
2. WeightEach question has a weight (1.0–1.3). Weighted average within dimension.
3. Dimension scoreWeighted avg of all questions in dimension → 0-100
4. Total scoreSimple avg of 8 dimension scores → 0-100
5. Maturity level0-25 INICIAL / 26-50 BASICO / 51-75 INTERMEDIARIO / 76-100 AVANCADO
6. Benchmark deltadimension_score - segment_p50 (from benchmark_cache)

Recommendation Formula

priority_score = (impact × 0.35) + (speed × 0.25) + (evidence × 0.25) + (efficiency × 0.15)

// threshold: score < 0.60 → suppressed (never shown to user)
// P0: score ≥ 0.85 | P1: ≥ 0.75 | P2: ≥ 0.65 | P3: ≥ 0.60
